{# SpringServe Adapter Connection Configuration Template Included in the settings page when the SpringServe adapter is selected. Supports two auth flows: email+password (canonical, auto-refreshing 2hr token) and pre-minted api_token (escape hatch). Exactly one of (email+password) or api_token must be provided. Expected context: - adapter_config: dict with current adapter configuration - tenant_id: current tenant ID #}

SpringServe (Magnite) Configuration

Direct connection to the SpringServe ad server -- the path that avoids Magnite's SSP-level AdCP agent and the fees that come with it. Provide the API user's email + password (recommended -- the adapter mints + refreshes the 2-hour token automatically), or paste a pre-minted token under Advanced.

Inventory Sync

Pull SpringServe supply partners + supply tags into the local cache. The product setup UI uses this so you can pick targeting from synced inventory instead of typing IDs. Today supply-side reads return 403 on most accounts -- ask SpringServe support to enable supply read scope.

API Permissions

Probe each SpringServe endpoint the adapter depends on to verify the configured credentials have the required scopes. Each check is a cheap read; nothing is mutated upstream.
